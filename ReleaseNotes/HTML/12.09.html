Siemplify Marketplace Release Notes - 12.09

Published on 12 May 2019

Actions

  • IntSights
    • Support new IntSights API

  • Cuckoo
    • Get Report
      • Added retry logic when fetching report failed
    • Detonate File
      • Fetched report attachment replaced with a link to an online report order to save space on the Siemplify machine
    • Detonate URL
      • Fetched report attachment replaced with a link to an online report order to save space on the Siemplify machine

  • Cisco Threat Grid
    • Get Report
      • Fetched report attachment replaced with a link to an online report order to save space on the Siemplify machine

  • Active Directory
    • Get Manager Details
      • Bug fixed - fail when manager not found

  • CA Service Desk
    • Wait For Status Change
      • Added support for Unicode

  • Crowdstrike Falcon (previously - Crowdstrike)
    • Integration name was changed from "Crowdstrike" to "Crowdstrike Falcon"
    • 'Get Device Details' name was changed to 'Get Host Information'
    • Added new actions
      • Update IOC information
      • Delete IOC
      • List uploaded IOCs
      • List Hosts
      • Get process name by IOC

  • MISP
    • Mark MISP suspicious events on Siemplify
      • Added support for Unicode

  • URLSCAN.IO
    • Added option to uncheck 'verify SSL certificate' on integration configuration - to solve some problems with Proxies
      • Added support for Unicode

  • MSSQL
    • Run SQL Query
      • Added support for INSERT, UPDATE and DELETE queries

    Connectors

    • LogRhythm Alarms Connector - New
      • Added new connector to ingest cases by alarm

    • Email Connector
      • Connector extended to support more email providers
      • Improved environment logic
        • Added 'Environment field name' parameter - define an environment field manually
        • Added RegEx pattern parameter to extract specific string from environment field
        • Added environment aliases file - in case the user need to map between environment raw field to Siemplify environment

    • Microsoft Exchange Connector
      • Improved environment logic
        • Added 'Environment field name' parameter - define an environment field manually
        • Added RegEx pattern parameter to extract specific string from environment field
        • Added environment aliases file - in case the user need to map between environment raw field to Siemplify environment

    • ArcSight Connector
    • Connector error log improved to reduce information logs
    • Bug fixed - Correlations retention not deleting files correctly

      • Qradar Connector
      • Improved environment logic
        • Added 'Use QRadar Environments' checkbox - ingest cases to Siemplify environment by Qradar multi-tenant mechanism
        • Added 'Environment field name' parameter - define an environment field manually
