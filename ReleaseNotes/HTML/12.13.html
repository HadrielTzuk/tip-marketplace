Siemplify Marketplace Release Notes - 12.13

Published on 23 June 2019

Actions

  • ElasticSearch
    • Simple ES Search
      • Parameters descriptions and default values were updated for better user experience
    • Advanced ES Search
      • Parameters descriptions and default values were updated for better user experience
      • Time frame parameter changed to >=, <= instead of >,<
  • ThreatQ
    • Test Connectivity - minor bug fix
  • Portnox
    • Integration configuration - "verify SSL" - minor bug fix
  • Twilio
    • Send SMS and Wait
      • Added "unique identifier code" mechanism to match each SMS response to it's originating action instance (in cases of multiple actions that happen simultaneously, or multiple replies)
        • Outgoing messages are now followed by "To respond type your reply followed by {unique identifier code}".
    • Ping
      • Test connectivity - minor bug fix
  • SSH
    • Run Command
      • "Remote port" parameter- minor bug fix
  • CaServiceDesk
    • Wait for Status Change
      • Added Json Result - the action querying the ticket and returning JsonResult when ticket status is changed
      • Improved logging and error handling
    • Search Tickets
      • Added option to search ticket by ID
        • If "Ticket ID" is set - ignore other fields
      • Fixed typo in parameter name: "Summery" -> "Summary" (regressive in playbooks)
  • Palo Alto Panorama
    • Added support for version 9
      • Configure your product version under integrartion configuration
  • SCCM
    • Actions descriptions and parameters descriptions were updated for better user experience
  • Cisco ISE
    • Self Signed Certificate and Proxy support bug fix
  • SEP
    • Scan Endpoint
      • "Scan type" parameter modified from "string" to "drop-down list"
  • Siemplify
    • Create Entity
      • Improved error handling - raise an error when one of multiple entities creation was failed
  • CrowdStrike Falcon
    • New parameters were added to the integration configuration to support new actions
      • Contain Endpoint - New Action
        • Quarantine an endpoint by Hostname or IP address
      • Lift Contained Endpoint - New Action
        • Unquarantine an endpoint by Hostname or IP address.
  • TenableSecurityCenter
    • Get Report - New Action
      • Retrieve a report by ID or name
  • CSV
    • CSV Search by Entity - New Action
      • Search for records in CSV file(s) by case entities and enrich entities accordingly
    • CSV Search by String - New Action
      • Search for records in CSV file(s) by keyword and return results
  • VirusTotal
    • Scan Hash
      • Entities are marked as suspicious if the number of engines which detected hash as malicious is equal or above the given threshold (in the past - entities marked above threshold only)
    • Scan URL
      • Entities are marked as suspicious if the number of engines which detected URL as malicious is equal or above the given threshold (in the past - entities marked above threshold only)

Jobs

  • Siemplify
    • Siemplify Measurement Monitor Improvements
      • Added option to save a copy of CSV report to a local folder for each job cycle. This can be configured using a new parameter - "Metrics Output Folder" (non mandatory).
      • Added option to configure how many CSV reports to keep backwards - "Max CSV Files Count Retention" (non mandatory). When exceeding the limit - older CSV reports will be overridden.
        • Any more files then configured will be deleted (oldest first)
    • Siemplify Log Collector
      • Fixed Log Duplication Bug
