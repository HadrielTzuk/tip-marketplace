Siemplify Marketplace Release Notes - 13.11

Published on 19 August, 2020

Integrations

  • MSSQL
    • MSSQL
      • Add support for using windows authentication with Linux version of Siemplify platform (using Kerberos authentication). Please refer to documentation for more information.
  • MicrosoftAzureSentinel
    • Microsoft Azure Sentinel
      • Refactored integration to support Python 3.7
  • OpswatMetadefender
    • Opswat Metadefender
      • Updated API Key to be encrypted.
      • Added "Verify SSL" parameter to the integration configuration.
  • Slack
    • Slack
      • Add support for pagination.

Actions

  • ActiveDirectory
    • Remove User From Group
      • NEW! New action for removing a user from a group.
    • Update attributes of an AD User
      • NEW! New action for updating attributes of existing Active Directory users.
    • Update attributes of an AD Host
      • NEW! New action for updating attributes of existing Active Directory hosts.
  • CheckPointFirewall
    • Add a Sam Rule
      • NEW! New "Add a Sam Rule" action added. Notice, the action performs a publish command after adding the rule.
  • MicrosoftDefenderATP
    • Enrich Entities
      • Fix a bug that prevented the action to properly enrich entities.
      • Add missing JSON results example.
    • Get File Related Alerts
      • Update action to return found alerts info as JSON results.
    • Get File Related Machines
      • Update action to return found machines info as JSON results.
    • Get Machine Logon Users
      • Update action to return found users info as JSON results.
    • Get Machine Related Alerts
      • Update action to return found alerts info as JSON results.
    • Update Alert
      • Update JSON result example to match the actual JSON that is returned by the action.
  • Siemplify
    • Raise Incident
      • Add a new "Soc Role" parameter.
  • Slack
    • Wait For Reply
      • Add new "Channel ID" parameter to support placeholders as input for the parameter.
    • List Channels
      • Fix a bug that prevented the action to display non-ASCII channel names.
      • Remove default value for "Max Channels to Return" parameter to allow the action to bring all channels if limit is not provided.

Connectors

  • MicrosoftAzureSentinel
    • Microsoft Azure Sentinel Incident Connector
      • Fix a bug that causes the connector to miss incidents in high volume environments.
      • Fix a bug that prevents the connector from setting the correct environment when "Environment Field Name" field is set.
  • PaloAltoCortexXDR
    • Palo Alto Cortex XDR Connector
      • Add "parent_start_time" and "parent_end_time" fields to all events.
  • Sumologic
    • Sumologic Connector
      • NEW! Modified connector to parse out the "_raw" field for each alert to allow the data to be mapped within Event Mapping settings.
