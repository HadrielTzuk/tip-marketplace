Siemplify Marketplace 12.15 Release Notes

Published on August 4th,  2019

 

RELEASE TECHNICAL DETAILS:

  • This marketplace version can only be installed on Siemplify 5.0 and up

 

Actions

 

  • LogRhythm

 

    • Removed unnecessary fields from integration configuration (TIP-766)
    • Test Connectivity
      • Fixed a bug caused by a missing dependency (TIP-766)

 

  • Palo Alto PANORAMA

 

    • Added descriptions for missing parameters (TIP-596)

 

  • Palo Alto NGFW

 

    • Added descriptions for missing parameters (TIP-596)
    • Fixed a bug caused by a missing dependency (TIP-727)
  • Email
    • Fixed bug in body extraction for plaintext / HTML emails
    • Added support for smart reply extraction for using in playbooks
      • Action will extract a string surrounded by special placeholder - {{ mystring }} and put into “JsonResult.answer” - Can be used in playbooks using the Expression Builder.
      • In case of multiple placeholders - action will use the first one only
    • Send email and wait (TIP-786)

 

  • Siemplify

 

    • Permitted Alert Time
      • Changed Start time, Endtime parameters to mandatory state (to prevent failure on execution) (TIP-729)
      • *Please note - this change is regressive - the user will have to update his playbooks.
  • Updated JsonResult Sample to the following integrations: (TIP-728)
    • Siemplify
    • Email
    • Exchange
    • MSSQL
    • MongoDB
    • Palo Alto Panorama
    • Palo Alto NGFW

 

Connectors

 

  • CBResponse - Connector

 

      • Changed multiple fields to a non-mandatory state (TIP-708)

 

  • Jira - Connector

 

      • Improved handling of empty fields in Jira events to reduce irrelevant data in Siemplify (TIP-725)
      • Extended support for non-UTC Jira servers when querying for tickets (TIP-725)

 

  • Email - Email Connector

 

 

        • Changed field names of emails’ “Filename”, “Filehash” and URLs to allow extraction of multiple values by the new Siemplify ontology algorithm  (TIP-706)

 

  • Email - EML Connector

 

        • Fixed a bug causing new emails not to be ingested into Siemplify (TIP-821)

 

  • Exchange - Mail Connector
    • Changed field names of emails “Filename”, “Filehash” and URLs to allow extraction of multiple values by the new Siemplify ontology algorithm  (TIP-706)

 

    • Fixed a bug causing new emails not to be ingested into Siemplify (TIP-821)

 

  • QRadar - Connector

 

    • Added validation of custom fields existence to prevent a potential bug in Qradar events fetching (TIP-717)

 

Jobs

 

  • Siemplify
    • Cases Collector

 

        • Fixed bug in processing cases collected from the publisher (TIP-795)
